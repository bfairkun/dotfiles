Use the `mcp__jupyter-kernel__connect_to_kernel` tool with no arguments to attach to the most recently modified Jupyter kernel connection file. This searches only the standard runtime dirs (`~/.local/share/jupyter/runtime/` and `/run/user/$UID/jupyter/runtime/`), so it reliably picks VS Code's current kernel rather than any Claude-started kernel in `/tmp/`. Then confirm which connection file was attached and run `import sys; print(sys.executable)` via `mcp__jupyter-kernel__run_python` to confirm the kernel is live.
